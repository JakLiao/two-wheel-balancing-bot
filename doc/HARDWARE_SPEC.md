## 三、HC-05 蓝牙模块接线

### 3.1 模块外观与 6 引脚排列

> HC-05 蓝牙模块共有 **6 个引脚**，从左到右（面对模块正面，邮票孔在右侧）排列如下：

```
┌─────────────────────────────────────────────────────────┐
│  ┌──────┐                                    ┌──────┐  │
│  │ [LED] │   HC-05 蓝牙模块（正面）          │      │  │
│  └──────┘                                    └──邮票─┘  │
│                                                         │
│   ①TXD   ②RXD   ③GND   ④VCC   ⑤STATE   ⑥EN       │
│                                                         │
│  正面视图：引脚从左到右依次排列                          │
└─────────────────────────────────────────────────────────┘
```

### 3.2 各引脚功能详解

| 引脚 | 名称 | 方向 | 功能说明 | 本项目接线 |
|------|------|------|---------|-----------|
| **①** | **TXD** | 输出→STM32 | UART 发送引脚，模块发出的数据 | → **PA3**（USART2_RX） |
| **②** | **RXD** | 输入←STM32 | UART 接收引脚，模块接收的数据 | → **PA2**（USART2_TX） |
| **③** | **GND** | 电源 | 模块地线，与 STM32 共地 | → **GND** |
| **④** | **VCC** | 电源 | 供电正极，范围 3.6V~6V（板载稳压芯片）| → **5V** |
| **⑤** | **STATE** | 输出→STM32 | 连接状态指示脚（可不用）：未连接=低电平，配对成功=高电平 | **可不接**（悬空）|
| **⑥** | **EN** | 输入 | 模块使能脚：接地=禁用，悬空/3.3V=使能 | **可不接**（悬空）|

### 3.3 接线示意图

```
HC-05 模块（6Pin）              STM32F103C8T6
━━━━━━━━━━━━━━━                 ━━━━━━━━━━━━━━━━
  TXD  ① ────────────────────   PA3  （USART2_RX）
  RXD  ② ────────────────────   PA2  （USART2_TX）
  GND  ③ ────────────────────   GND
  VCC  ④ ────────────────────   5V   （3.6V~6V 均可）
  STATE⑤ ────────────────────   （可不接，悬空即可）
  EN   ⑥ ────────────────────   （可不接，悬空即可）
```

### 3.4 关键注意事项

- **TXD/RXD 交叉接线**：HC-05 的 TXD → STM32 的 RX（PA3），HC-05 的 RXD → STM32 的 TX（PA2）
- **电平兼容**：HC-05 使用 3.3V TTL（可与 5V STM32 直连，无需电平转换）
- **VCC 范围**：3.6V~6V，**禁止超过 7V**，否则会烧模块
- **最少接线数**：实际使用只需接 **4 根线**（TXD、RXD、GND、VCC），STATE 和 EN 平时悬空即可
- **波特率**：默认 9600，配对后可配置为 115200（代码中设为 115200）
- **配对密码**：默认 1234

### 3.5 STATE 引脚说明（可选）

STATE 引脚用于指示蓝牙连接状态，可用来驱动 LED 或接回 STM32 判断连接状态：

| HC-05 状态 | STATE 引脚电平 | LED 现象 |
|-----------|--------------|--------|
| 未配对/未连接 | 低电平（0V）| LED 快闪（2Hz）|
| 配对成功/通信中 | 高电平（3.3V）| LED 慢闪（0.5Hz）或常亮 |

如需使用，在模块上找一个 STATE 焊盘（通常在 LED 旁边）引出即可。

### 3.6 EN 引脚说明（通常不用）

| EN 状态 | 模块行为 |
|--------|--------|
| 悬空或接 3.3V | 模块正常工作（默认）|
| 接地 | 模块关闭，蓝牙功能禁用 |

> 正常使用时不需连接此引脚，保持悬空即可。

## 四、MPU6050 模块接线

```
MPU6050 模块                    STM32F103C8T6
━━━━━━━━                       ━━━━━━━━━━━━━━━━
  VCC  ───────────────────────  3V3（或 5V）
  GND  ───────────────────────  GND
  SCL  ───────────────────────  PB10（I2C2_SCL）
  SDA  ───────────────────────  PB11（I2C2_SDA）
  ADO  ───────────────────────  PB9（地址位，接地=0x68）
  INT  ───────────────────────  PB8（外部中断，数据就绪下降沿）
```

> **注意：** MPU6050 模块一般自带 4.7kΩ 上拉电阻，无需外部再加。

### ADO 引脚说明

- **ADO 接地（PB9 输出低）** → 从地址 = **0x68**（常用）
- ADO 接 VCC（PB9 输出高）→ 从地址 = 0x69
- 代码默认使用 **0x68**，与 ADO 接地配合

### INT 引脚说明

- MPU6050 数据就绪时 INT 引脚输出**低电平脉冲**
- PB8 配置为**外部中断**（EXTI），下降沿触发
- 中断服务函数中读取 `MPU6050_REG_ACCEL_XOUT_H` 等寄存器

## 七、PID 控制框架

### 两环串联 PID（与 `src/app/balance.c` 注释一致）

```
遥控期望速度 ──→ [速度环 PID] ──→ 期望倾角(或倾角偏置)──→ [直立环 PID] ──→ PWM
 ↑ ↑
 编码器测速 MPU 俯仰角(互补滤波)
```

| 环 | 输入 | 输出 | 目标 |
|----|------|------|------|
| **速度环(外环,较慢)** | 轮速(编码器,脉冲/s)与期望速度差 | 期望倾角 / 倾角偏置 | 跟踪速度、抗推回中 |
| **直立环(内环,较快)** | 俯仰角与(速度环给出的)目标角差 | 电机 PWM / 力矩指令 | 车体保持直立 |

> **整定顺序**：先关速度环或置零期望，只调直立环使车能立住；再逐步打开速度环。细节见 `doc/PID_GUIDE.md`。

## 十、编码器 6Pin 线接线定义

> 来源：电机参数.xlsx（2026-05-19）

### 线序颜色定义

| 颜色 | 功能 | 说明 |
|------|------|------|
| **红** | 电机电源 + | → TB6612FNG VM（12V），交换红/白可控制电机正反转 |
| **白** | 电机电源 - | → TB6612FNG GND |
| **蓝** | 编码器 VCC（正） | → STM32 3.3V 或 5V |
| **黑** | 编码器 GND（负） | → STM32 GND（必须与 STM32 共地） |
| **黄** | A 相信号 | → STM32 对应 TIM 通道（每圈 11 PPR） |
| **绿** | B 相信号 | → STM32 对应 TIM 通道（每圈 11 PPR） |

### STM32 具体引脚连接

| 轮 | A相（黄） | B相（绿） | TIM | 编码器分辨率 |
|----|---------|---------|-----|------------|
| **左轮** | PA15 | PB3 | TIM2_CH1 / TIM2_CH2 | 224 PPR × 4 = 896 CPR/圈 |
| **右轮** | PB6 | PB7 | TIM4_CH1 / TIM4_CH2 | 224 PPR × 4 = 896 CPR/圈 |

### 共地规则（重要）

> STM32 GND、编码器黑线、TB6612 GND、12V 电源负极 **四者必须连在一起**，否则 I2C 和编码器信号会不稳定。

### 让两轮编码器极性统一的机械安装原则

两轮平衡车左右轮面对面安装，电机轴朝向相反 → 前进时两编码器原始极性必然相反。

**解法：统一电机轴朝向**

```
俯视图（车轮朝纸面外）

    左轮           右轮
   [电机A]       [电机B]
    轴→            轴→      ← 两只电机轴都朝向车头方向
```

机械上统一后，两轮前进时编码器都输出**正数**，软件不需要取反处理。

## 十一、编码器计数对应关系

| 电机 | TIM | 引脚 | 计数方向 |

| 左电机 | TIM2 | PA15(A) / PB3(B) | 四倍频，方向自动识别 |
