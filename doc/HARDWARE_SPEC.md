## 一（附）、你提供的 TB6612 排线顺序（待与模块丝印核对）

你给出的连接器顺序为：

`GND, B1, B14, B13, A7, A4, A5, B0, GND, B02, B01, A02, A01, GND, 5V, VM`

**解读原则**：上表为“线序 → MCU 引脚名”，**不等价于 TB6612 官方引脚名**。请务必用万用表对照你手上的 TB6612 模块丝印，将 **PWMA/AIN1/AIN2/PWMB/BIN1/BIN2/STBY** 与上述 GPIO 一一对应后再上电。

**与 MPU6050 的潜在冲突（重要）**

- 你同时写明 **MPU6050 接 PA4、PA5、PA7、PA6**，而线序中也出现 **PA4、PA5、PA7** 接驱动相关信号。  
- **同一 GPIO 不能既做 I2C/中断又做电机方向或 PWM**，否则 MPU 与电机必有一处错误。

**推荐做法（软件/教学最省心）**

- **MPU6050**：使用硬件 I2C1 **PB6=SCL、PB7=SDA**（或 Remap 到 **PB8/PB9**，以便把 **PB6/PB7 让给 TIM4 编码器**）。INT 可接任意空闲 EXTI（如 PB12，视板子走线而定）。  
- **TB6612**：PWM 用 **TIM1：PA8（CH1）、PA9（CH2）**；方向脚用空闲 GPIO（如 PB3~PB5 等在关闭 JTAG 后释放的脚，注意与超声波 PB3 冲突时换脚）。  
- **双编码器**：**TIM2：PA0/PA1** + **TIM4：PB6/PB7**（或与 I2C Remap 组合，详见 `doc/SYSTEM_DESIGN.md` §三）。  
- **HC-SR04**：你计划 **Trig=PA15、Echo=PB3** → 必须在 Cube 中选 **Serial Wire Debug**，关闭 JTAG 以释放 PA15、PB3、PB4；若 PB3 仍与调试或其他外设冲突，可将 Echo 改到其他定时器可捕获引脚。

## 二、引脚分配表（推荐方案：与 Keil/Cube 工程一致后冻结）

下列为**推荐且与本仓库 `pin_map.h` 默认一致**的一套接法（若你改版布线，请同步改 `pin_map.h` 与 Cube 配置）。

### 2.1 TB6612FNG 电机驱动

| TB6612 引脚 | STM32 引脚 | 功能 |
|-------------|-----------|------|
| PWMA | PA8（TIM1_CH1） | 左轮 PWM 调速 |
| AIN1 | PB5 | 左轮方向控制 1 |
| AIN2 | PB4 | 左轮方向控制 2 |
| PWMB | PA9（TIM1_CH2） | 右轮 PWM 调速 |
| BIN1 | PB8 | 右轮方向控制 1 |
| BIN2 | PB9 | 右轮方向控制 2 |
| STBY | PB0 | 驱动总使能（必须接高才能输出） |
| GND | GND | 共地 |
| VM | 7.4V 锂电池正极 | 电机电源 |

### 2.2 编码器电机（JGA25-370 或同类）

| 电机引脚 | STM32 引脚（推荐） | 功能 |
|---------|-------------------|------|
| 左轮 A 相 | PA0（TIM2_CH1） | 左轮编码器 A |
| 左轮 B 相 | PA1（TIM2_CH2） | 左轮编码器 B |
| 右轮 A 相 | PB6（TIM4_CH1） | 右轮编码器 A（**需 TIM4 编码器模式**） |
| 右轮 B 相 | PB7（TIM4_CH2） | 右轮编码器 B |
| 编码器电源 | 3.3V | 编码器供电 |
| 电机 GND | GND | 共地 |

> ⚠️ 编码器侧务必确认模块是 **3.3V 还是 5V 逻辑**；若为 3.3V 霍尔，接 5V 可能损坏。  
> ⚠️ **STM32 编码器模式仅使用各定时器的 CH1+CH2**；不能用“单定时器 CH3+CH4”冒充第二路正交解码。  
> 若右轮仍使用 PA2/PA3，则应配置 **TIM3 等另一组 TI1/TI2**，并处理与 USART2 等默认复用冲突。

### 2.3 MPU6050 姿态传感器（I2C）

| MPU6050 | STM32 引脚 | 功能 |
|---------|-----------|------|
| SDA | PB7 或 PB9 | I2C 数据（与编码器、Remap 统一规划） |
| SCL | PB6 或 PB8 | I2C 时钟 |
| VCC | 3.3V | 供电 |
| GND | GND | 共地 |
| INT（可选） | 任意 EXTI | 数据就绪中断（第一版可轮询省略） |

### 2.4 HC-05 蓝牙模块（USART，引脚待确认）

典型接法（**交叉接线**）：

| HC-05 | STM32 | 说明 |
|-------|-------|------|
| VCC | 5V | 模块主供电 |
| GND | GND | 共地 |
| TXD | MCU USART_RX | 蓝牙发 → MCU 收 |
| RXD | MCU USART_TX | MCU 发 → 蓝牙收（若模块非 3.3V 容忍需在 RX 端串电阻或电平转换，以模块手册为准） |
| KEY / EN（若有） | 悬空或按键 | AT 模式配置波特率、名称时用 |

**示例（与仓库默认一致）**：USART3，**PB10=TX、PB11=RX**（MCU 侧接 HC-05 的 RXD/TXD 交叉）。

### 2.5 HC-SR04 超声波（可选）

| HC-SR04 | STM32（你的计划） | 备注 |
|---------|------------------|------|
| Trig | PA15 | 需 **SWD 调试**、关闭 JTAG |
| Echo | PB3 | 同上；建议输入脚配定时器输入捕获测脉宽，或外部中断+定时器测宽 |
| VCC | 5V | 常见模块 5V |
| GND | GND | 共地 |

## 二（附）、JGA25-370 速度换算

速度环需要“脉冲/采样周期 → 轮速”。请从供应商资料读取：**编码器线数、减速比、是否 AB 相 4 倍频**。  
在代码中设置与 `encoder.c` 中一致的**每转脉冲数**常量，否则速度环 PID 量纲错误。

## 三、共地规则（重中之重）

> 所有模块 GND 全部连在一起，漏接必翻车。

```
STM32 GND
  = TB6612 GND
  = MPU6050 GND
  = HC-05 GND
  = 电池负极
```

## 五、PID 控制框架

### 两环串联 PID（与 `src/app/balance.c` 注释一致）

```
遥控期望速度 ──→ [速度环 PID] ──→ 期望倾角（或倾角偏置）──→ [直立环 PID] ──→ PWM
                      ↑                                         ↑
                 编码器测速                               MPU 俯仰角（互补滤波）
```

| 环 | 输入 | 输出 | 目标 |
|----|------|------|------|
| **速度环（外环，较慢）** | 轮速（编码器，脉冲/s）与期望速度差 | 期望倾角 / 倾角偏置 | 跟踪速度、抗推回中 |
| **直立环（内环，较快）** | 俯仰角与（速度环给出的）目标角差 | 电机 PWM / 力矩指令 | 车体保持直立 |

> **整定顺序**：先关速度环或置零期望，只调直立环使车能立住；再逐步打开速度环。细节见 `doc/PID_TUNING_GUIDE.md`。
