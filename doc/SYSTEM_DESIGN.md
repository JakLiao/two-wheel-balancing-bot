# 两轮平衡车 · 系统设计方案（第一版）

> 主控：野火双 USB STM32F103C8T6  
> 姿态：MPU6050，**第一版仅用一阶互补滤波**（不用卡尔曼、不用 DMP）  
> 执行：TB6612FNG + JGA25-370（双霍尔编码器）  
> 通信：HC-05；测距：HC-SR04（可选，平衡第一版可不参与控制）

---

## 一、总体架构

平衡车控制本质是**小车上倒立摆**：以俯仰角为被控量，通过轮子的加速度产生支撑力矩。

```
                    ┌─────────────────┐
  遥控/期望速度 ──→ │ 速度环 PID      │──→ 期望倾角 offset
                    └────────┬────────┘
                             ▼
                    ┌─────────────────┐
  俯仰角(互补滤波) ─│ 直立环 PID      │──→ 左右轮 PWM（差速可实现转向）
                    └────────┬────────┘
                             ▼
                    TB6612 → 电机
                             │
         编码器 ────────────┘（速度反馈，速度环用）
```

**整定顺序（与 `doc/PID_GUIDE.md` 一致）**

1. 先**直立环**：车体在原地能大致立住（可先关速度环，用手轻扶找感觉）。  
2. 再**速度环**：叠加遥控期望，使车能“跟速”、被推后能回中。  
3. **转向**：在左右轮速度指令上加差速（或单独偏航目标），与直立环输出叠加后限幅。

**第一版刻意省略**

- 卡尔曼滤波、扩展卡尔曼：对 F103 + 教学场景非必须；互补滤波足够验证机械与 PID。  
- MPU DMP：固件复杂、黑盒多，不利于理解传感器与滤波。  
- 超声波参与平衡：HC-SR04 更新慢、有盲区，建议 M2 以后仅用于避障/停车，不进入直立环主路径。

## 二、姿态估计：一阶互补滤波（不用卡尔曼）

### 2.1 物理含义

- **加速度计**：静态倾角准，动态受振动、线加速度干扰大。  
- **陀螺仪**：短期角速度准，对角度积分会漂移。

### 2.2 融合公式（俯仰角 θ，离散周期 dt）

记陀螺仪俯仰角速度为 `gy`（°/s），加速度计算出的俯仰角为 `angle_acc`（°）。一阶互补：

\[
\theta_k = \alpha \cdot (\theta_{k-1} + g_y \cdot dt) + (1-\alpha) \cdot angle\_{acc}
\]

工程常用 **α = 0.96 ~ 0.99**（与仓库 `mpu6050.c` 中 `COMPLEMENTARY_ALPHA` 同量级）：  
- α 偏大：更信陀螺，跟手但易受陀螺零偏影响。  
- α 偏小：更信加速度计，静态稳但车体抖动时角度会“粘”在加速度噪声上。

`angle_acc` 由三轴加速度在“侧立安装、俯仰绕车轴”的坐标系下用 `atan2` 计算（具体轴与安装方向需在标定小节一致，见硬件文档）。

### 2.3 实现要点

- 控制周期建议 **5 ms（200 Hz）** 或 **10 ms（100 Hz）**；陀螺积分项使用**与控制周期相同的 dt**。  
- 上电后先延时数百毫秒再读 MPU，避免上电冲击。  
- 第一版可不做磁力计/偏航闭环；**偏航漂移**不影响“前后平衡”主任务。

## 三、双编码器与定时器：为何需要“4 个 TIM 通道”

STM32F1 的**编码器模式**只接在定时器的 **TI1 / TI2**（对应 CH1、CH2），**一个通用定时器只能接一路正交编码器**。

因此两路电机（A、B 相各一根）共 **4 根编码器线**，在 Cube/HAL 里表现为：

| 轮子   | MCU 引脚（示例，见硬件规格书最终表） | 定时器资源      |
|--------|--------------------------------------|-----------------|
| 左轮 AB| 如 PA15 / PB3                      | TIM2 CH1 / CH2  |
| 右轮 AB| 如 PB6 / PB7 或 PA2 / PA3 等       | TIM4 CH1 / CH2（或 TIM3 等） |

**注意**：仓库中 `encoder.c` 曾注释“左右轮应用独立定时器”；**TIM2 的 CH3/CH4 不能替代第二路正交解码**，第二路必须占用 **另一个** TIMx 的 CH1+CH2。  
Keil/Cube 工程里请配置 **TIM2 Encoder + TIM4 Encoder**（或 TIM3，视引脚与 I2C 重映射而定）。

JGA25-370 的**每转脉冲数**与减速比、磁极数有关，务必查该电机供应商数据表，在速度环里换算“脉冲/秒 → mm/s 或 rpm”。

## 四、实时任务划分（建议）

| 周期     | 任务 |
|----------|------|
| 200 Hz   | MPU 读数、互补滤波、直立环 PID、PWM 更新 |
| 100 Hz   | 编码器采样算速、速度环 PID（可略低于直立环） |
| 10~20 Hz | 蓝牙指令解析、状态机 |
| 10 Hz    | HC-SR04 触发/回波（若启用），勿阻塞 200 Hz 控制 |

使用 SysTick 或 TIM 基准触发，**避免在 I2C 轮询里长时间阻塞**；若后续换 DMA + 中断可再优化。

## 五、电源与共地

- **TB6612 VM**：接电池（如 2S 7.4V），大电流走粗线。  
- **TB6612 VCC**：模块逻辑电源，按模块说明接 3.3V 或 5V（以模块手册为准）。  
- **STM32、MPU6050**：3.3V；**MPU6050 若为 5V 容忍模块仍建议只供 3.3V** 以免损坏裸芯片。  
- **HC-05**：典型 5V 供电；与 MCU 串口电平需确认模块是否为 3.3V TTL（多数 HC-05 可与 3.3V MCU 直连，以模块说明为准）。  
- **所有模块 GND 汇于一点**，电机动力地与信号地尽量在电池/TB6612 星形汇流，减少电机 PWM 对模拟传感器的干扰。
