# 两轮平衡车 · PID 实用整定指南

> 面向实际操作，硬件验证完成后使用
> 参考：`doc/PID_TUNING_GUIDE.md` · 源码：`src/app/balance.c`

---

## 1. 整定前的准备

### 硬件状态确认（必须先通过串口测试）

在开始调 PID 之前，先用串口助手确认以下状态，全部通过再上车：

- [ ] MPU6050 输出稳定角度值（静止时 pitch ≈ 0 ± 2°）
- [ ] 编码器在车轮静止时 rpm ≈ 0（无噪声跳变）
- [ ] TIM3 PWM 手动设置占空比 50 时电机平稳运转
- [ ] 电池电压充足（> 11V），避免低压导致 PID 异常

### 安全防护

- 车体上电前用手扶住，做好扶手准备
- 准备一块软垫做"倒地缓冲"
- 串口助手打开，观察数据

## 2. 控制环说明（与 balance.c 对应）

### 直立环（核心，200 Hz）

| 项目 | 说明 |
|------|------|
| 输入 | 俯仰角 pitch（°） |
| 输出 | PWM（-100 ~ +100） |
| 目标 | 车体垂直 |
| PID 参数 | `BALANCE_KP / BALANCE_KI / BALANCE_KD` |
| 默认值 | KP=50, KI=0, KD=20 |

### 速度环（外环，100 Hz）

| 项目 | 说明 |
|------|------|
| 输入 | 编码器速度（脉冲/s） |
| 输出 | 期望倾角 offset（°） |
| 目标 | 跟踪遥控速度 |
| PID 参数 | `SPEED_KP / SPEED_KI / SPEED_KD` |
| 默认值 | KP=1.0, KI=0.01, KD=0 |

## 3. 整定步骤

### Step 1：直立环（最重要）

**目标：** 让车体能大致立住（哪怕不稳定）

**方法：**

1. 先把 `BALANCE_KP` 从 0 开始慢慢加（每次 +5）
2. 观察：车体有"硬"的感觉，KP 太大会剧烈抖动
3. 找到车体开始振荡的临界值，记录
4. 加入 `BALANCE_KD`（建议从 KP/5 开始），抑制振荡
5. 调好的标志：用手轻推车体，车体有阻力但不会立刻倒

**判断标准：**

| 现象 | 原因 | 处理 |
|------|------|------|
| 车体软，推倒无阻力 | KP 太小 | 继续加 KP |
| 车体能短时间立住，被推后能弹回 | KP 合适 | 记录此值 |
| 车体剧烈抖动/啸叫 | KP 太大 | 退回上一步，减 10% |

---

### Step 2：速度环（让车能跟速、不跑远）

**前置条件：** 直立环调好，车体能稳定立住

**目标：** 车体能被推动后回到原点

**方法：**

1. 直立环调好后，把 `SPEED_KP` 从 0 开始加（每次 +0.5）
2. 观察：推动车体后，车体是否能回中
3. `SPEED_KI` 消除稳态偏移（积分有滞后，少量即可）

**判断标准：**

| 现象 | 原因 | 处理 |
|------|------|------|
| 推动后不回中 | SPEED_KP 太小 | 加 KP |
| 松手后慢慢回中，无振荡 | 正常 | — |
| 松手后快速回弹再振荡 | SPEED_KI 太大 | 减 KI |

## 4. 参数参考范围（基于 balance.c 默认值）

| 参数 | 默认值 | 安全范围 | 说明 |
|------|--------|---------|------|
| BALANCE_KP | 50 | 20 ~ 150 | 越大越"硬"，太大会抖 |
| BALANCE_KI | 0 | 0 | 通常不用，平衡车不需要 |
| BALANCE_KD | 20 | 5 ~ 80 | 抑制振荡，越大越阻尼 |
| SPEED_KP | 1.0 | 0.1 ~ 5.0 | 跟手感 |
| SPEED_KI | 0.01 | 0 ~ 0.1 | 消除静差，过大会振荡 |
| SPEED_KD | 0 | 0 | 速度环一般不用 D |

### 直立环典型调参路径（参考）

```
KP=10  → 车身有反应但立不住，推一下就倒
KP=30  → 车体能立住几秒，推一下能回正一点
KP=50  → 车体立得较稳，推一下能弹回来  ← 默认值
KP=80  → 车身开始抖（振荡临界）
KP=60  → 回到比较稳的状态  ← 推荐起始点

KD=5   → 振荡略微减小
KD=15  → 推一下，车体能快速回正，振荡消失
KD=30  → 车身变得"僵硬"，响应变慢
KD=20  → 良好平衡点  ← 默认值
```

### 速度环典型调参路径（参考）

```
SPEED_KP=0.5  → 手推车，车速有轻微回正趋势
SPEED_KP=1.0 → 遥控前进/后退，车速跟踪较好  ← 默认值
SPEED_KP=2.0 → 速度响应快但松手后有明显回弹振荡

SPEED_KI=0.01 → 稳态误差减小  ← 默认值
SPEED_KI=0.05 → 松手后速度快速归零，效果良好
SPEED_KI=0.1  → 出现振荡
```

## 5. 常见问题排查

| 现象 | 原因 | 解决方案 |
|------|------|---------|
| 车体一直往一边倒 | 机械不平衡 / 角度零点漂移 | 重新校准 MPU6050 零点 |
| PWM 一直满幅输出 | MPU6050 数据全为 0（I2C 不通） | 检查 I2C2 总线 |
| 低速抖动 | 编码器计数错误 | 检查编码器接线 |
| 车身振荡不止 | KD 太小 | 加大 KD |
| 直立环 P 很小就抖 | MPU6050 共地不良 / 电机驱动干扰 | 检查 GND 连线，加滤波电容 |
| 速度环加不上就抖 | 直立环不够稳 | 先把直立环调好再加速度环 |
| 编码器数值乱跳 | 电机电源干扰 / 编码器线太长 | 编码器信号线加 100Ω 电阻 |
| 车倒了电机还在转 | 倾角保护阈值太大 | 缩小 BALANCE_ANGLE_MAX（balance.c 中 pitch > ±45° 保护） |
| 手机指令和实际方向相反 | 电机接线或方向定义反了 | 调换 AIN1/AIN2 或修改代码 |

## 6. 调参小技巧

### 串口实时绘图

在主循环打印一行数据，用 SerialPlot 实时画图：

```c
printf("%+.2f,%+.2f,%d,%d\r\n", pitch, gyro, left_speed, right_speed);
```

### 蓝牙动态调参（免重烧）

让 HC-05 接收特殊指令动态修改 PID 参数，不用来回烧代码：

```
SET_BALANCE_KP=60\r\n
SET_BALANCE_KD=25\r\n
GET_STATUS\r\n
```

### 示波器查 PWM

用示波器直接测 PA8/PA9，看 PWM 波形是否正常输出。

## 7. 安全注意事项

1. **上车前先绑绳子**，防止车跑出去摔坏
2. **手放在车旁边**，随时准备扶住
3. **先低电压（3.7V）测试**，确认正常再上 7.4V
4. **倾角超过 45° 自动停电机**，防止电机烧毁（balance.c 中已实现）
5. **不要长时间空转电机**，TB6612FNG 会发热
