## 二、PWM 量程换算说明

> ⚠️ 本节是理解 PID 参数数值的关键前提

本车代码 `BALANCE_OUT_MAX=100`，PID 输出 `±100` 直接写入 TIM3 compare 寄存器（范围 0~99）。

野火官方 STM32 平衡车标准 PWM 量程是 **0~7199**（ARR=7199，PSC=0，10kHz），安全输出 **±6000**，死区补偿 **300~400**。

**量程换算系数：**

```
换算系数 = 100 / 7200 ≈ 0.0139
```

**江协科技参考值换算：**

> 江协科技参考 Kp=200~700 基于 PWM 量程 7200（ARR=7199）

```
Kp 参考范围（换算到本车量程）：
  最小：200 × 0.0139 ≈ 2.8
  最大：700 × 0.0139 ≈ 9.7

→ 江协科技换算推荐：Kp = 2.8 ~ 9.7
```

**死区补偿说明：**

野火标准中 TB6612FNG 电机有 300~400 PWM 计数的死区。本车 ARR=99，死区比例换算：

```
本车死区阈值 = 350 × (99/7199) ≈ 5（compare 寄存器值）
```

由于本车 PWM 量程仅 0~99，无法直接套用野火的死区补偿方法。当前代码用 ±100 作为中间量直接写入 compare 寄存器，死区问题由 `BALANCE_KP` 足够大来掩盖。如需改善低速性能，建议在 `Motor_Set_Speed()` 中判断 `|speed| < 5` 则直接 `Motor_Stop()`。

## 三、Ziegler-Nichols 临界比例度法推导

### Step 1：系统固有频率估算（倒立摆简化模型）

```
等效摆长 h ≈ 35 cm = 0.35 m（重心到轮轴）

ω₀ = √(g/h) = √(9.8 / 0.35) ≈ 5.29 rad/s
T₀ = 2π/ω₀ ≈ 1.19 s
```

### Step 2：系统转动惯量估算

```
车身质量 m ≈ 1.5 kg

绕轮轴的转动惯量（简化圆柱体模型）：
I = (1/3) × m × h² = (1/3) × 1.5 × 0.35² ≈ 0.061 kg·m²
```

### Step 3：电机力矩常数换算

```
电机规格：起动转矩 2.6 kg·cm = 0.255 N·m（输出轴）
减速比：20.4 : 1

电机轴等效扭矩常数（估算）：
Kt = 0.255 N·m / 2.67A ≈ 0.0955 N·m/A

PWM=100 时的最大输出扭矩（输出轴）：
T_max = 0.255 N·m

等效推进力：
F_max = T_max / r = 0.255 / 0.03 = 8.5 N
车身重量 ≈ 15 N → 最大加速度 ≈ 0.57g
```

### Step 4：临界增益 Ku 估算

```
阻尼比 ζ ≈ 0.10 ~ 0.20（欠阻尼，直立平衡车自然特征）
取 ζ = 0.15

ω_damped = ω₀ × √(1-ζ²) ≈ 5.29 × 0.988 ≈ 5.23 rad/s

临界增益（归一化到 PWM 量程 100）≈ 6.4

Ziegler-Nichols 临界比例度法参数表（欠阻尼修正）：
┌────────┬──────────┬───────────┬───────────┐
│  类型   │    Kp    │    Ki     │    Kd     │
├────────┼──────────┼───────────┼───────────┤
│   P    │  Ku/2    │     0     │     0     │
│   PI   │  Ku/2.2  │  Ku/8.8   │     0     │
│  PID  │  Ku/1.5  │  Ku/3.3   │  Ku/2.6   │
└────────┴──────────┴───────────┴───────────┘

→ Z-N PID 参考值：Kp≈6.4, Ki≈1.9, Kd≈2.5
```

## 五、Kp/Kd/Ki 过大过小判断标准

> 来源：江协科技 + 物理推导双重验证

### Kp（比例增益）

| 状态 | 现象 | 应对 |
|------|------|------|
| **Kp 过小** | 推一下，车慢慢回正（响应迟钝）或不回正倒地 | ↑ 增大 Kp |
| **Kp 合适** | 推一下 → 车快速回正，1~2 次内停 | ✅ 最优 |
| **Kp 偏大** | 低频来回摆动（0.5~2Hz），2~3 次才停 | ↓ 减小 Kp |
| **Kp 过大** | 持续低频振荡，越来越厉害，直到倒地 | 立刻 ↓ 减小 Kp |

### Kd（微分增益）

| 状态 | 现象 | 应对 |
|------|------|------|
| **Kd 过小** | 推一下后，来回振荡 3 次以上才停 | ↑ 增大 Kd |
| **Kd 合适** | 推一下 → 回正干脆，无多余振荡 | ✅ 最优 |
| **Kd 偏大** | 静止时有细微高频颤动，电机有轻微啸叫 | ↓ 减小 Kd |
| **Kd 过大** | 剧烈高频抽搐（⚠️会损伤电机），立刻断电 | 立刻 ↓ 减小 Kd |

### Ki（积分增益）

| 状态 | 现象 | 应对 |
|------|------|------|
| **Ki=0** | 无积分，稳态误差无法消除 | 直立环通常 Ki=0 即可 |
| **Ki 过小** | 稳态时轮子有明显持续低速运动 | 调大 Ki |
| **Ki 合适** | 松手后车能缓慢回到中立点 | 当前 Ki=0 ✅ |

**速度环 Kp 判断标准：**

- SPEED_KP 太小：遥控前进指令发了，车速跟不上，响应迟钝
- SPEED_KP 合适：遥控前进 → 车速快速跟踪指令
- SPEED_KP 太大：遥控一停，车立刻猛刹（有顿挫感）

## 六、调参前硬件状态确认

> ⚠️ **机械问题 > PID 调参**。硬件没调好，PID 再怎么调都是徒劳。

### 🔴 第一步：拆除所有软体负载

**毛绒玩偶（或任何软体/晃动负载）是导致 PID 调参失败的头号原因。**

- 软体负载动态晃动，PID 算法无法补偿这种随机、不可预测的运动
- 大幅抬高整车重心，转动惯量急剧增大，PID 难度指数级上升
- 前后重量分布不均，机械中立点偏差过大

**正确顺序：**
1. 完全拆除毛绒玩偶
2. PID 调稳后，再用热熔胶+扎带 **100% 固定死**（不能有任何晃动）
3. 重新微调参数

### 机械准备

1. **拆除所有软体/可晃动负载**（毛绒玩偶等）
2. **电池放到最低位置**，尽量贴近两个轮子连线（电池是最重部件，决定重心）
3. **核心板和扩展板水平安装**，不能歪斜
4. **所有接线用扎带固定**，避免晃动
5. **左右重量尽量对称**
6. **粗略找机械中立点**：把车放在手指上，前后轻轻推动，找到能稳住 1-2 秒的位置

### 串口确认步骤

打开串口助手（115200 8N1），确认以下全部通过后再上车调 PID：

| 检查项 | 期望现象 | 如不符合 |
|--------|---------|---------|
| `[Balance] angle_offset=...` | 打印出零点偏置值 | MPU6050 未初始化完成 |
| `P=`（pitch） | 静止时 ≈ 0° ± 3°（已自动去除偏置） | `balance_angle_offset` 采样异常 |
| `Gx/Gy/Gz` | 静止时 ≈ 0 °/s | 需重新校准 MPU6050 |
| `Ax/Ay/Az` | Az ≈ 16384，其他轴 ≈ 0 | I2C 通信异常 |
| 编码器 RPM | 静止时 ≈ 0.0 | 编码器信号跳变，需先排查 |
| 电池电压 | > 11V | 电量不足会导致 PID 异常 |
| `angle_offset` 的值 | 通常 ±3° 以内 | 过大说明传感器初始角度偏差大 |

```c
#define SPEED_KD        0.0f

#define SPEED_OUT_MAX   30.0f
#define SPEED_OUT_MIN  -30.0f
```

### 安全阈值

```c
#define BALANCE_ANGLE_MAX  33.0f
```

### 双环串联 PID 架构

```
期望速度（遥控）──→[速度环 PID]──→ target_angle（期望倾角）──→[直立环 PID]──→电机 PWM
                      100Hz                              200Hz
                      输出 ±30°                          输出 ±100
```

### 直立环 PID 核心公式

```c
float target_angle = -speed_output;
float final_output = PID_Calculate(&balance_pid, target_angle, pitch, 0.005f);
```

**`PID_Calculate` 内部（增量式 PID）：**

```c
float error = target_angle - measurement;
float p_out = kp * error;
pid->integral += ki * error * dt;
float dedt = (error - prev_error) / dt;
float d_out = kd * dedt;
output = p_out + integral + d_out;
```

### 直立环物理意义（结合代码）

以「车后仰 + 遥控前进」为例：

```
情况：车后仰 pitch=+5°，遥控前进 speed_output=+10

step1: target_angle = -10（速度环输出取负）
step2: error = -10 - 5 = -15
step3: p_out = Kp × (-15)
step4: d_out = Kd × dedt（抑制倾斜变化率）
step5: output → clamp → 电机正转，车轮前进 → 纠正后仰 ✅
```

## 八、直立环调参步骤（5步）

> ⚠️ 正式开始前，确认硬件状态检查全部通过（见第六节）

### Step 1：确认 D 极性（先加微分项）

**为什么先加 D？** D 项的作用是抑制倾斜变化率，相当于"提前刹车"。如果 D 极性反了，车会越推越抖，而不是被阻尼住。

1. 手扶车，车轮离地或用东西垫起来
2. 慢慢往前推车头——车轮应该**抵抗推力**，而不是顺着推力转
3. 如果车轮顺着推力转（无阻尼）→ D 极性可能有问题
4. 把车往后推——车轮也应该**抵抗**，方向相反

### Step 2：P 从极小值开始找临界震荡点

**目标：找到 Kp 临界值（车开始低频来回摆动的点）**

从 Kp=1.0 开始，每次 +0.5，观察现象：

| Kp | 现象 | 下一步 |
|----|------|--------|
| 1.0 | 有轻微阻力感 | → 继续加 |
| 2.0 | 阻力明显 | → 继续加 |
| **4.0~6.0** | **开始出现低频抖动（类似"点头"，0.5~2Hz）** | → 记录临界点 |
| 8.0+ | 低频抖动明显，来回摆动 | → 超过临界 |

**找到临界 Kp**（车刚开始出现低频来回摆动的值），例如 `Kp_critical = 6.0`

### Step 3：取临界值 × 0.6 作为初始 Kp

```
工程 Kp = Kp_critical × 0.6
例如：临界Kp = 6.0 → 6.0 × 0.6 = 3.6 → 取 3.5~4.0
```

### Step 4：逐步加 Kd 消振

**目标：消除低频抖动，让车推一下能快速回正但不来回摆。**

| Kd | 现象 | 下一步 |
|----|------|--------|
| 0.05 | 低频抖动仍然存在 | → 加 Kd |
| 0.10 | 抖动略微减小 | → 继续加 |
| **0.12~0.20** | **推一下能快速回正，无明显振荡** | **← 重点区间** |
| 0.30+ | 高频颤抖出现（电机啸叫）| → 立刻减小 |

**判断标准：**
- 推一下，车体有阻力且能在 1~2 次内回正 → Kd 合适
- 来回振荡 2 次以上才停 → Kd 太小
- 高频颤抖（电机啸叫）→ Kd 太大

### Step 5：微调 & 记录

**调好的标志：**
- 用手轻推车体，车体有阻力但不会立刻倒
- 推一下后能在 1~2 次内回正（不超过 3 次）
- 无高频颤抖，无持续低频振荡

## 九、速度环调参（直立环调好后进行）

> ⚠️ 务必确认直立环调好后再动速度环参数。速度环叠加在直立环上，调乱会影响直立环表现。

**前置条件：直立环调好，车体能稳定立住 5 秒以上**

```
SPEED_KP = 0.0    → 手推车，车速无回正趋势
SPEED_KP = 0.5    → 有轻微回正趋势
SPEED_KP = 1.0    → 遥控前进/后退，车速跟踪较好 ← 推荐起始值
SPEED_KP = 2.0    → 响应快但松手后有明显回弹振荡

SPEED_KI = 0.005  → 稳态误差减小 ← 当前值
SPEED_KI = 0.01   → 松手后速度快速归零
SPEED_KI = 0.05   → 出现振荡 → 退回
```

**判断标准：**
- 遥控前进，车速能跟上指令 → Kp 合适
- 松手后车慢慢停下（而不是立刻停）→ Ki 合适
- 速度响应迟钝 → Kp/Ki 太小
- 来回加减速振荡 → Kp/Ki 太大

## 十、Z-N 临界比例度法实车操作步骤

> 适配本车量程的操作流程

```
第一步：准备
  → 拆除所有软体负载，固定重心
  → Ki=0, Kd=0, Kp=0（最小值）

第二步：逐步增大 Kp，找临界振荡点
  → 从 Kp=1.0 开始，每次 +0.5
  → 观察：车开始出现等幅低频来回摆动（不倒但不停）
  → 记录这个 Kp 值 = Kp_critical

第三步：按 Z-N 表格求参数
  直立环用 PID 档：
    Kp = Kp_critical × 0.6
    Ki = Kp_critical × 0.4 / 100（换算到本车量程）
    Kd = Kp_critical × 0.075

第四步：上车验证，微调
  → 如果还有振荡 → Kd 微调 +0.02
  → 如果响应迟钝 → Kp 微调 +0.5
```

| `SET_SPEED_KP=1.0\r\n` | 设置速度环 Kp |
| `SET_SPEED_KI=0.005\r\n` | 设置速度环 Ki |
| `GET_STATUS\r\n` | 查询所有当前参数 |
| `DUMP\r\n` | 打印所有 PID 参数值 |

**串口解析代码示例（app_balance.c）：**

```c
#include <string.h>
#include <stdlib.h>

void Balance_Parse_Command(char *cmd) {
    if (strncmp(cmd, "SET_BALANCE_KP=", 15) == 0) {
        float val = atof(cmd + 15);
        if (val >= 0 && val <= 500) {
            balance_pid.kp = val;
            printf("[PID] BALANCE_KP=%.1f\r\n", val);
        }
    } else if (strncmp(cmd, "SET_BALANCE_KD=", 15) == 0) {
        float val = atof(cmd + 15);
        if (val >= 0 && val <= 10) {
            balance_pid.kd = val;
            printf("[PID] BALANCE_KD=%.3f\r\n", val);
        }
    } else if (strcmp(cmd, "DUMP") == 0) {
        printf("[PID] BALANCE Kp=%.1f Ki=%.2f Kd=%.3f\r\n",
               balance_pid.kp, balance_pid.ki, balance_pid.kd);
        printf("[PID] SPEED   Kp=%.2f Ki=%.3f Kd=%.2f\r\n",
               speed_pid.kp, speed_pid.ki, speed_pid.kd);
    }
}
```

## 十二、常见问题排查

| 现象 | 原因 | 解决 |
|------|------|------|
| 🔴 车一直往一边倒，不是在中立点来回摆 | **软体/晃动负载**未拆除 | 拆除所有软体负载，固定所有晃动部件 |
| P 极性反了，车往一边倒 | P 极性反了 | 改为负值或调换 AIN1/AIN2 |
| Kp 很小就开始抖 | MPU6050 共地不良 / 电机干扰 | 检查 GND，编码器加 100Ω 电阻 |
| 低频来回摆动（>2次才停）| Kp 太大或 Kd 太小 | ↓Kp 或 ↑Kd |
| 高频颤抖（电机啸叫）| Kd 太大 | ↓Kd |
| 推一下才倒，无阻尼 | Kp 太小 | ↑Kp |
| 直立环稳了但速度环一加就抖 | 速度环加太急 | 先把 SPEED_KP 降回 0，逐步加 |
| 编码器数值乱跳 | 电机电源干扰 / 线太长 | 编码器信号线换短，加 100Ω 电阻 |
| 车倒了电机还在转 | 倾角保护阈值太大 | 检查 `pitch > 33.0f` 判断条件 |
| 手机指令和实际方向相反 | 电机接线或方向定义反了 | 调换 AIN1/AIN2 或修改 Motor_Differential |
| `angle_offset` 打印值 > ±10° | 启动时车体不在竖直位置 | 重新上电，保持车体垂直再启动 |
| PWM 一直满幅输出 | MPU6050 数据全为 0（I2C 不通） | 检查 I2C2 总线 |
| 车轮不动，pitch=0 但电机无输出 | 倾角太小（未达到启动阈值）| 检查保护阈值是否生效 |
