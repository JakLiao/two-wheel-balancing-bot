## ⚠️ 阅读前必看（v2.1 更新）

**本版本根据实际调车反馈进行了关键修正，以下是核心变化：**

### 🔴 第一优先级：拆除所有软体负载

**毛绒玩偶（或任何软体/晃动负载）是导致 PID 调参失败的头号原因。**

- 软体负载动态晃动，PID 算法无法补偿这种随机、不可预测的运动
- 大幅抬高整车重心，转动惯量急剧增大，PID 难度指数级上升
- 前后重量分布不均，机械中立点偏差过大

**正确顺序：**
1. 完全拆除毛绒玩偶
2. PID 调稳后，再用热熔胶+扎带 **100% 固定死**（不能有任何晃动）
3. 重新微调参数

### ⚠️ P=160 量程换算说明（v2.2 核心修正）

| 参数 | 旧默认值 | **v2.2 推荐起始值** |
|------|---------|-------------------|
| `BALANCE_KP` | `160.0f` | **`0.3~1.0`** |
| `BALANCE_KD` | `0.1f` | **`0.02~0.1`** |

**完整换算过程（重要）：**

本车代码 `BALANCE_OUT_MAX=100` 意味着 PID 输出 `±100` 直接写入 TIM3 compare 寄存器（范围 0~99）。

野火官方 STM32 平衡车标准 PWM 量程是 **0~7199**（ARR=7199，PSC=0，10kHz），安全输出 **±6000**，死区补偿 **300~400**。

**量程换算基准（理论参考）：**

```
换算系数 = 99 / 7199 ≈ 0.0137
野火 Kp=18~25 → 本车基准 Kp ≈ 0.25~0.35
```

> ⚠️ **换算基准 ≠ 实测最优值**：上述 0.25~0.35 是「机械完全相同」假设下的结果。本车加装了毛绒玩偶等额外重量，机械更重，**实测最优 Kp=10~20**（约为基准值的 40~80 倍）。

> 💡 **操作建议**：以实测最优值 **10~20** 为参考起点，Kd 从 **`0.5~1.0`** 开始微调。

### 🔧 死区补偿说明

野火标准中 TB6612FNG 电机有 **300~400 PWM 计数**的死区（PWM < 死区值时电机不转）。

本车 ARR=99，野火 ARR=7199，死区比例换算：
```
本车死区阈值 = 350 × (99/7199) ≈ 5（compare 寄存器值）
```

> ⚠️ **重要说明**：由于本车 PWM 量程仅 0~99，无法直接套用野火的死区补偿方法（加 350 会溢出 compare 寄存器）。
> 当前代码用 ±100 作为中间量直接写入 compare 寄存器，死区问题由 `BALANCE_KP` 足够大来掩盖（输出 > 死区阈值 5 时才有效）。
> 如需改善低速性能，建议在 `Motor_Set_Speed()` 中判断 `|speed| < 5` 则直接 `Motor_Stop()`（刹车）。

## 一、调参前硬件状态确认

> ⚠️ **机械问题 > PID 调参**。硬件没调好，PID 再怎么调都是徒劳。

### 🔴 第一步：机械准备（必须先做）

1. **拆除所有软体/可晃动负载**（毛绒玩偶等），见前述说明
2. **电池放到最低位置**，尽量贴近两个轮子连线（电池是最重部件，决定重心）
3. **核心板和扩展板水平安装**，不能歪斜
4. **所有接线用扎带固定**，避免晃动
5. **左右重量尽量对称**
6. **粗略找机械中立点**：把车放在手指上，前后轻轻推动，找到能稳住 1-2 秒的位置

### 确认步骤

打开串口助手（115200 8N1），确认以下全部通过后再上车调 PID：

| 检查项 | 期望现象 | 如不符合 |
|--------|---------|---------|
| `[Balance] angle_offset=...` | 打印出零点偏置值 | MPU6050 未初始化完成 |
| `P=`（pitch） | 静止时 ≈ 0° ± 3°（已自动去除偏置） | `balance_angle_offset` 采样异常 |
| `Gx/Gy/Gz` | 静止时 ≈ 0 °/s | 需重新校准 MPU6050 |
| `Ax/Ay/Az` | Az ≈ 16384，其他轴 ≈ 0 | I2C 通信异常 |
| 编码器 RPM | 静止时 ≈ 0.0 | 编码器信号跳变，需先排查 |
| 电池电压 | > 11V | 电量不足会导致 PID 异常 |
| `[Balance] angle_offset=...` 的值 | 通常 ±3° 以内 | 过大说明传感器初始角度偏差大 |

**正常启动打印示例：**
```
[MPU6050] Found device at 0x68, WHO_AM_I=0x68 (OK)
=========================================
  Balance Bot FW Started!
  SYSCLK: 72MHz, UART: 115200 8N1
=========================================
[Balance] angle_offset=0.34 deg (avg of 30 samples)
P=-0.02 gx=0.0 | acc_pitch=0.07 | Ax=-10 Ay=19 Az=16385 Gx=2 Gy=-3 Gz=-2
```

> 注意：`P=` 值在平衡姿态时应 ≈ 0，而不是原始的 ±3°。因为代码在 `Balance_Init()` 时会自动采样 30 次求均值，存入 `balance_angle_offset`，后续所有 `pitch` 值都会减去这个偏置。

## 三、控制架构解析（代码逐行）

理解代码是调参的前提。先把架构看清楚，调参时才不会"不知道为什么车往那边跑"。

### 双环串联 PID

```
期望速度（遥控）──→[速度环 PID]──→ target_angle（期望倾角）──→[直立环 PID]──→电机 PWM
                      100Hz                              200Hz
                      输出 ±30°                          输出 ±100
```

### 直立环 PID 核心公式

```c
// speed_output：速度环输出 = 期望倾角（度）
float target_angle = -speed_output;

// 直立环 PID：让 pitch 跟踪 target_angle
float final_output = PID_Calculate(&balance_pid, target_angle, pitch, 0.005f);
```

**`PID_Calculate` 内部（增量式 PID）：**

```c
float error = target_angle - measurement;   // 期望 - 实际
float p_out = kp * error;
pid->integral += ki * error * dt;           // 积分（带限幅）
float dedt = (error - prev_error) / dt;     // 微分
float d_out = kd * dedt;
output = p_out + integral + d_out;
```

### 直立环物理意义（结合代码）

以「车后仰 + 遥控前进」为例：

```
情况：车后仰 pitch=+5°，遥控前进 speed_output=+10

step1: target_angle = -10（速度环输出取负）
step2: error = -10 - 5 = -15
step3: p_out = 1.0 × (-15) = -15
step4: d_out = 0.1 × dedt（抑制倾斜变化率）
step5: output ≈ -15 → clamp → -15（PWM 有效值）
step6: 电机正转，车轮前进 → 纠正后仰 ✅
```

**关键：速度环输出是"期望倾角"，直立环的 job 是让"实际倾角"追"期望倾角"。**

## 四、直立环调参步骤（5步）

> ⚠️ 正式开始前，确认硬件状态检查全部通过（见第一节）

### Step 1：确认 D 极性（先加微分项）

**为什么先加 D？** D 项的作用是抑制倾斜变化率，相当于"提前刹车"。如果 D 极性反了，车会越推越抖，而不是被阻尼住。

1. 准备：手扶车，车轮离地或用东西垫起来
2. 慢慢往前推车头——车轮应该**抵抗推力**，而不是顺着推力转
3. 如果车轮顺着推力转（无阻尼）→ D 极性可能有问题（极少数情况）
4. 然后把车往后推——车轮也应该**抵抗**，方向相反

> D 极性在绝大多数情况下不需要改，但调参前确认一次能减少很多误判。

### Step 2：P 从 0（或极小值）开始找临界震荡点

**目标：找到 Kp 临界值（车开始低频来回摆动的点）**

> ⚠️ 本车 PID 输出 ±100 直接写入 compare 寄存器（范围 0~99）。野火参考 Kp=18~25 对应本车约 **0.25~0.35**。
> **建议从 Kp=0.3 开始往上加**，每次 +0.1~0.2，观察现象。

```c
// v2.2 推荐起始值（野火 PWM 量程换算）
BALANCE_KP = 0.3f;    // 从这里开始，不从 0
BALANCE_KD = 0.02f;   // 从这里开始，不从 0
```

| Kp | 现象 | 下一步 |
|----|------|--------|
| 0.3 | 有轻微阻力感 | → 继续加 |
| 0.5 | 阻力明显 | → 继续加 |
| **1.0~2.0** | **开始出现低频抖动（类似"点头"，0.5~2Hz）** | → 记录临界点 |
| 3.0+ | 低频抖动明显，来回摆动 | → 超过临界 |

**找到临界 Kp**（车刚开始出现低频来回摆动的值），例如 `Kp_critical = 1.5`

### Step 3：取临界值 × 0.6 作为初始 Kp

```
工程 Kp = Kp_critical × 0.6
例如：临界Kp = 1.5 → 1.5 × 0.6 = 0.9 → 取 0.8~1.0
```

> 临界值通常在 1.0~2.0 附近（对应野火参考 Kp=18~25 换算后的值），以实测为准。

### Step 4：逐步加 Kd 消振

**目标：消除 Step 2 中的低频抖动，让车推一下能快速回正但不来回摆。**

```c
// 以 Step 3 得到的 Kp 为基础（例：Kp=0.9）
BALANCE_KP = 0.9f;    // Step 3 结果
BALANCE_KD = 0.02f;   // 从推荐起始值开始
```

| Kd | 现象 | 下一步 |
|----|------|--------|
| 0.02 | 低频抖动仍然存在 | → 加 Kd |
| 0.05 | 抖动略微减小 | → 继续加 |
| **0.08~0.15** | **推一下能快速回正，无明显振荡** | **← 重点区间** |
| 0.2+ | 高频颤抖出现（电机啸叫）| → 立刻减小 |

**判断标准：**
- 推一下，车体有阻力且能在 1~2 次内回正 → Kd 合适
- 来回振荡 2 次以上才停 → Kd 太小
- 高频颤抖（电机啸叫）→ Kd 太大
- 车身僵硬迟钝 → Kd 太大

### Step 5：微调 & 记录

```c
// 最终参考参数（v2.2 量程换算值）
#define BALANCE_KP  0.8~1.0    // 临界值 × 0.6
#define BALANCE_KI  0.0f
#define BALANCE_KD  0.8~1.2  // 视振荡情况调整
```

**调好的标志：**
- 用手轻推车体，车体有阻力但不会立刻倒
- 推一下后能在 1~2 次内回正（不超过 3 次）
- 无高频颤抖，无持续低频振荡

## 五、速度环调参（直立环调好后进行）

> ⚠️ 务必确认直立环调好后再动速度环参数。速度环叠加在直立环上，调乱会影响直立环表现。

**前置条件：直立环调好，车体能稳定立住 5 秒以上**

```c
// 速度环默认值（通常不需要大改）
#define SPEED_KP  1.0f
#define SPEED_KI  0.01f   // 积分消除静差
#define SPEED_KD  0.0f
```

**调参步骤：**

```
SPEED_KP = 0.0    → 手推车，车速无回正趋势
SPEED_KP = 0.5    → 有轻微回正趋势
SPEED_KP = 1.0    → 遥控前进/后退，车速跟踪较好 ← 默认值
SPEED_KP = 2.0    → 响应快但松手后有明显回弹振荡

SPEED_KI = 0.01   → 稳态误差减小 ← 默认值
SPEED_KI = 0.05   → 松手后速度快速归零
SPEED_KI = 0.1    → 出现振荡 → 退回
```

**判断标准：**
- 遥控前进，车速能跟上指令 → Kp 合适
- 松手后车慢慢停下（而不是立刻停）→ Ki 合适
- 速度响应迟钝 → Kp/Ki 太小
- 来回加减速振荡 → Kp/Ki 太大

## 六、串口调参界面（免重烧录）

每次改参数重烧录太麻烦，建议通过串口指令在线修改参数。

**协议（文本格式，方便调试）：**

| 指令 | 说明 |
|------|------|
| `SET_BALANCE_KP=70\r\n` | 设置直立环 Kp |
| `SET_BALANCE_KD=0.1\r\n` | 设置直立环 Kd |
| `SET_SPEED_KP=1.0\r\n` | 设置速度环 Kp |
| `SET_SPEED_KI=0.01\r\n` | 设置速度环 Ki |
| `GET_STATUS\r\n` | 查询所有当前参数 |
| `DUMP\r\n` | 打印所有 PID 参数值 |

**在 `app_balance.c` 中添加串口解析（示例）：**

```c
#include <string.h>
#include <stdlib.h>

void Balance_Parse_Command(char *cmd) {
    if (strncmp(cmd, "SET_BALANCE_KP=", 15) == 0) {
        float val = atof(cmd + 15);
        if (val >= 0 && val <= 500) {
            balance_pid.kp = val;
            printf("[PID] BALANCE_KP=%.1f\r\n", val);
        }
    } else if (strncmp(cmd, "SET_BALANCE_KD=", 15) == 0) {
        float val = atof(cmd + 15);
        if (val >= 0 && val <= 10) {
            balance_pid.kd = val;
            printf("[PID] BALANCE_KD=%.3f\r\n", val);
        }
    } else if (strcmp(cmd, "DUMP") == 0) {
        printf("[PID] BALANCE Kp=%.1f Ki=%.2f Kd=%.3f\r\n",
               balance_pid.kp, balance_pid.ki, balance_pid.kd);
        printf("[PID] SPEED   Kp=%.2f Ki=%.3f Kd=%.2f\r\n",
               speed_pid.kp, speed_pid.ki, speed_pid.kd);
    }
}
```

**使用方式：**
1. 用蓝牙串口模块（HC-05/08）连接手机或电脑
2. 发送 `SET_BALANCE_KP=80` 即可实时修改参数
3. 配合 `DUMP` 确认修改成功

## 八、常见问题排查

| 现象 | 原因 | 解决 |
|------|------|------|
| 🔴 车一直往一边倒，不是在中立点来回摆 | **软体/晃动负载**（毛绒玩偶）未拆除 | 拆除所有软体负载，固定所有晃动部件 |
| P 从 20 开始加但车一直往一边倒 | P 极性反了 | 改为负值或调换 AIN1/AIN2 |
| Kp 很小（<20）就开始抖 | MPU6050 共地不良 / 电机干扰 | 检查 GND，编码器加 100Ω 电阻 |
| 低频来回摆动（>2次才停）| Kp 太大或 Kd 太小 | ↓Kp 或 ↑Kd |
| 高频颤抖（电机啸叫）| Kd 太大 | ↓Kd |
| 推一下才倒，无阻尼 | Kp 太小 | ↑Kp |
| 直立环稳了但速度环一加就抖 | 速度环加太急 | 先把 SPEED_KP 降回 0，逐步加 |
| 编码器数值乱跳 | 电机电源干扰 / 线太长 | 编码器信号线换短，加 100Ω 电阻 |
| 车倒了电机还在转 | 倾角保护阈值太大 | 检查 `pitch > 33.0f` 判断条件 |
| 手机指令和实际方向相反 | 电机接线或方向定义反了 | 调换 AIN1/AIN2 或修改 Motor_Differential |
| `angle_offset` 打印值 > ±10° | 启动时车体不在竖直位置 | 重新上电，保持车体垂直再启动 |
| 车轮不动，pitch=0 但电机无输出 | 倾角太小（未达到启动阈值）| 检查保护阈值是否生效 |
| 直立环一直调不稳，抖动剧烈 | **PWM 量程换算问题**：Kp=160 对 0~99 量程过大 | 把 Kp 降到 **10~20**（实测最优区间）|
